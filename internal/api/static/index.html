The Problem

Anti-bot systems block your agents

Every headless browser gets caught. Cloudflare, PerimeterX, DataDome, Akamai — they all see through Playwright and Puppeteer. Your agent hits a CAPTCHA wall and gives up.

Problem 01

TLS fingerprint detection

Cloudflare's server-side checks compare your browser's TLS ClientHello against known headless signatures. No amount of JavaScript tricks can fake a real TLS fingerprint. Aperture uses Camoufox — a modified Firefox that passes.

Problem 02

Canvas & WebGL fingerprinting

Anti-bot systems draw hidden canvases and compare the output. Headless Chrome renders identically every time — a dead giveaway. Aperture injects canvas noise, blocks WebRTC, and randomizes viewports on every session.

Problem 03

Behavioral analysis

Real humans don't click at pixel-perfect coordinates in 0ms. Bot detectors track mouse paths, scroll patterns, and timing. Aperture uses Bézier curve humanization with natural jitter and timing variation.

Architecture

The 4-layer stack

Your agent sends a goal. Aperture plans, stealth-hardens, executes, and auto-escalates to Camoufox if blocked. One MCP call, zero config.

Layer 1
Agent Brain Not us
Claude, GPT, Gemini, or your custom model. Decides what to do. Not our problem.
Layer 2
Aperture Engine ← We are here
LLM planner decomposes goals into steps. Stealth module hardens each browser instance. 5-browser pool with pre-warmed sessions.
Layer 3
Anti-Bot Detection ← We are here
Auto-detects Cloudflare, PerimeterX, DataDome, Akamai. Native Chrome for clean sites, Camoufox fallback for hardened ones. Zero manual config.
Layer 4
The Hostile Web The enemy
Anti-bot walls, CAPTCHAs, TLS fingerprinting, behavioral analysis, IP reputation scoring. We crack it so your agents don't have to.

Data Extraction

Every page becomes structured data

Describe what you want. Aperture navigates, renders the page (bypassing anti-bot if needed), and extracts typed JSON via LLM — no selectors, no DOM parsing.

◎

Schema-based extraction

Define your schema in plain English: "Extract product name, price, and rating." Get typed JSON back. The LLM handles the HTML.

⚡

Works on protected sites

Extract data from Cloudflare-protected, PerimeterX, and DataDome sites. Aperture handles the anti-bot bypass before extraction.

↻

Scoped extraction

Target specific page sections with CSS selectors. Extract from a table, a sidebar, or the main content — not the entire page noise.

∞

JSON or Markdown

Choose your output format. JSON for structured pipelines. Markdown for agent context. Both go through the same LLM extraction.

Developer Experience

One call. Any site. Structured data.

Use Aperture via MCP tools (for AI agents), REST API (for apps), or curl (for testing). No SDK required.

MCP Tool Call via OpenClaw / Claude 
// Agent calls the "extract" MCP tool
{
  "tool": "extract",
  "arguments": {
    "url": "https://news.ycombinator.com",
    "schema": "Top 5 posts with title, score, and author",
    "format": "json"
  }
}

// Or use "execute" for multi-step tasks
{
  "tool": "execute",
  "arguments": {
    "goal": "Go to GitHub trending and screenshot the page",
    "url": "https://github.com/trending"
  }
}
curl REST API on port 8080 
# Screenshot any URL
curl -X POST http://localhost:8080/api/v1/actions/screenshot \
  -H 'Content-Type: application/json' \
  -d '{"url":"https://producthunt.com"}'

# Multi-step browser task with LLM planning
curl -X POST "http://localhost:8080/api/v1/bridge/execute?sync=true" \
  -H 'Content-Type: application/json' \
  -d '{
    "goal": "Navigate to Amazon, search for headphones, extract top 3 results with prices",
    "url": "https://amazon.com"
  }'

# Health check
curl http://localhost:8080/api/v1/bridge/health
